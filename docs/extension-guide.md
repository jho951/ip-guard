# Extension Guide

## 새 규칙 타입 추가

새 IP 규칙 타입을 추가할 때는 `ip-guard-core` 안에서 판정 모델부터 추가합니다.

- 규칙은 `IpRule` 구현으로 표현합니다.
- 판정에 필요한 입력은 `IpAddress`와 `IpFamily` 수준에서 유지합니다.
- 파싱 책임은 `RuleParser`에 둡니다.
- 새 규칙이 기존 규칙과 충돌하지 않는지 테스트를 추가합니다.

## 새 문법 추가

룰 텍스트 문법을 확장할 때는 다음 순서를 따릅니다.

1. `RuleParser`에서 새 문법을 인식합니다.
2. `IpGuardEngine`이 새 문법을 포함한 규칙 집합을 처리하는지 확인합니다.
3. 경계값과 잘못된 입력을 함께 검증합니다.

권장 원칙:

- 문법은 가능한 단순하게 유지합니다.
- 같은 의미를 여러 표기법으로 허용하지 않습니다.
- 모호한 입력은 거부합니다.

## 새 판정 이유 추가

판정 결과를 더 자세히 설명해야 하면 `Decision`의 reason 값을 추가합니다.

- reason은 사람이 읽기 쉬워야 합니다.
- reason은 구현 세부사항보다 판정 결과를 설명해야 합니다.
- reason 추가 시에는 기존 테스트가 깨지지 않는지 확인합니다.

## 검증

새 확장 추가 후에는 최소한 아래를 확인합니다.

- 모듈 간 컴파일이 깨지지 않는지
- `IpGuardEngine`의 기본 판정이 바뀌지 않는지
- 입력/출력 모델이 불필요하게 복잡해지지 않는지
